# Role Manual: Project Manager

## Role Overview
The **Project Manager** is responsible for the successful delivery of specific projects. Your focus is on milestones, deadlines, and project-specific documentation.

## Permissions
- Manage project-specific teams and members.
- Create and manage Project Chat Groups.
- Share project-related files (Specs, Designs, CAD).
- Assign tasks and monitor completion status.

## Key Workflows

### 1. Project Launch
- Create a dedicated **Group** for the new project.
- Invite all relevant stakeholders and contributors.
- Upload the initial Project Charter and Timeline.

### 2. Task Assignment
- Use the chat to assign specific tasks to members.
- Use **Pinned Messages** for current milestones.

### 3. Documentation Control
- Manage the central repository of project files within the group.
- Ensure that the latest versions of technical documents are always available.

## Daily Usage Example
- Review the progress of the "Q3 Deployment" project group.
- Upload a 500MB video walk-through of the new project requirements.
- Update the pinned project timeline to reflect a new deadline.
