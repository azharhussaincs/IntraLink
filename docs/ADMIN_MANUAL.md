# ADMIN MANUAL

## Role Overview
The **Admin** is the highest-level user in the system. Admins are responsible for the overall system configuration, security policies, and top-level user management.

## Permissions
- Access to the Global Admin Dashboard.
- Create, Edit, and Deactivate any user account.
- Assign and change User Roles (promote to Super User, Team Lead, etc.).
- View Audit Logs for all system activities.
- Manage system-wide storage settings and backups.
- Configure organization-wide security policies (e.g., password complexity, session timeouts).

## Workflows

### 1. Onboarding a New Department
- Create a **Super User** (if one doesn't exist for the company/org).
- Create a **Team Lead** for the new department.
- Coordinate with the Team Lead to ensure they have the necessary permissions to build their teams.

### 2. Security Incident Response
- If a security breach is suspected, the Admin can:
    - View **Audit Logs** to track unauthorized access.
    - Immediately **Deactivate** suspicious accounts.
    - Revoke all active sessions.

### 3. Storage Maintenance
- Monitor the **Analytics Dashboard** for storage usage.
- If storage exceeds 80% capacity, coordinate with the System Administrator to expand physical storage or archive old files.

## Daily Usage Examples
- Reviewing system health alerts in the Admin Panel.
- Approving requests for new group creation (if restricted).
- Resetting a Team Lead's password who got locked out.
