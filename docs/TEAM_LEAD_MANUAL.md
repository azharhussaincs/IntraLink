# TEAM LEAD MANUAL

## Role Overview
The **Team Lead** (Department Head) is responsible for managing a specific department or large team. They bridge the gap between executive oversight and daily operations.

## Permissions
- Create and manage Teams.
- Create **Team Managers** and **Project Managers**.
- Create and manage department-wide chat groups.
- Send daily/weekly reports to the Super User.
- Monitor all activity within their assigned teams.

## Workflows

### 1. Team Structuring
- Create a new Team within the Department.
- Assign a **Team Manager** to handle the operational aspects.
- Add **Team Members** to the team.

### 2. Reporting
- At the end of each day/week, navigate to the **Reporting Module**.
- Fill out the mandatory report fields (Progress, Blockers, File Stats).
- Submit the report for Super User review.

## Daily Usage Examples
- Checking in with Team Managers via the "Leads & Managers" group.
- Reviewing file sharing activity to ensure department-level quotas are met.
- Resolving inter-team blockers identified in daily reports.
